## Henry the Bear in polyForth

Designed to run in a Volatco System:

* VOL00-01d - Volatco
* VOL01-00a - USB/Power

## Quick Start

To transform C#, or another structured language, into the block-oriented
polyFORTH used here:

1. Read the transform guide:
   [guides/CSHARP-TO-POLYFORTH.md](/home/cartheur/ame/aiventure/aiventure-github/emotional-toys/henry-bear-polyforth/guides/CSHARP-TO-POLYFORTH.md)
2. Follow the block-source rules:
   [guides/POLYFORTH-BLOCK-STANDARD.md](/home/cartheur/ame/aiventure/aiventure-github/emotional-toys/henry-bear-polyforth/guides/POLYFORTH-BLOCK-STANDARD.md)
3. Use the local reference model:
   [guides/IDEAL-LIST-BLOCKS.txt](/home/cartheur/ame/aiventure/aiventure-github/emotional-toys/henry-bear-polyforth/guides/IDEAL-LIST-BLOCKS.txt)
4. Produce the three output artifacts:
   - working source: [henry-face-motors.fth](/home/cartheur/ame/aiventure/aiventure-github/emotional-toys/henry-bear-polyforth/henry-face-motors.fth)
   - block source: [henry-face-motors-4800.src](/home/cartheur/ame/aiventure/aiventure-github/emotional-toys/henry-bear-polyforth/henry-face-motors-4800.src)
   - list capture: [guides/HENRY-FACE-LIST-BLOCKS.txt](/home/cartheur/ame/aiventure/aiventure-github/emotional-toys/henry-bear-polyforth/guides/HENRY-FACE-LIST-BLOCKS.txt)
5. Keep longer commentary out of the block source and place it in
   annotations:
   [guides/HENRY-FACE-ANNOTATIONS.md](/home/cartheur/ame/aiventure/aiventure-github/emotional-toys/henry-bear-polyforth/guides/HENRY-FACE-ANNOTATIONS.md)
